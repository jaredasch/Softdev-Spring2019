Issues with Regular Binary Trees

While regular binary trees can run most operations in O(logn) time (n is the number of elements in the tree), this is an oversimplification of its actual time complexity for most operations, which is O(h), where h is the height of the tree.

If elements are inserted into a binary tree in a certain order, the tree will essentially turn into a linked list, making its height the number of elements. This pushes the time complexity of most operations from O(logn) to O(n), a major decrease in efficiency.

Self Balancing Trees & Splay Trees

Because of the way in which binary trees work, there can be more than one configuration of a binary tree for the same data set, with different elements at the root. Through rearranging the tree in different ways, different properties can be acheived.

Splay trees adjust the structure of the tree such that after every Insert or Get operation, the most recently accessed node is at the root of the tree.

How Does it Work?

Tree adjustments are done with left and right rotations, which change the structure of the tree, including the root, while maintaining the properties of a binary tree

Right and left rotations are the buidling blocks of the more advanced transformations that are required for splay trees.

Advantages of Using Splay Trees

Much faster to access recently accessed elements

O(logn) amortized worst case time complexity – an improvement over regular binary search trees, which have a worst case time complexity of O(n)

Repeatedly accessing the same element after an initial access will run in O(1) time

Doesn’t require extra space for storing balancing information (Red-black trees, AVL trees)
